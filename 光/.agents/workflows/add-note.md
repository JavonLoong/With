# 工作流：添加知识笔记

当用户要求“调研/添加/记录 xxx 知识点到网站”时，你必须且只能按照以下精确的步骤执行：

## 步骤 1：深度调研与架构构思
对用户提供的知识点进行深度全网检索或第一性原理思考。
在书写之前，必须构思一个**“极度通俗的生活化比喻（白居易式一言蔽之）”**。

## 步骤 2：生成底层 Markdown 原文件
读取网站标准模板 @[d:\虚拟C盘\光\website\src\content\0-template.md] 的规范结构。
在 `d:\虚拟C盘\光\website\src\content\` 目录下新建一个文件，命名规则为：`[最新最大整数ID+1]-[英文短横线连字符].md`（例如：`9-docker-fundamentals.md`）。

文件必须包含标准的 YAML 头和三段式骨架：
**⚠️ 致命警告：绝对禁止照抄 `0-template.md` 里面原有的演示 `date`（比如 2026-03-2x 10:00 这种旧时间）！你必须调用系统时钟，填入你当前所处现实世界的真实时间，否则这篇文章会被永远沉入海底！**
```markdown
---
id: [自增ID]
title: [你的文章标题]
date: "[当前现实世界的真实时间 YYYY-MM-DD HH:mm]"
tags:
  - [相关标签1]
  - [相关标签2]
summary: [一句话总结痛点]
---

> **【一言蔽之】**
> [使用极端通俗的生活比喻或粗糙的白话解释这个概念的本质。让不懂技术的人也能瞬间建立直觉。]

---

## 🔬 一、 核心原理解析 (理解层)
[给出学术界/工程界的严谨定义、公式与第一性原理拆解]

## 🛠️ 二、 工程实践指南 (实践层)
[展示具体的代码、操作步骤、或系统架构落地图]

## 💡 三、 避坑与进阶 (升华层)
[点出初学者极易踩坑的误区、边界条件或同类技术对比]
```

## 步骤 3：触发自动化构建与部署
**注意：绝对不要去手动修改任何 JSON 文件！**
直接使用命令行执行部署：
```bash
cd "d:\虚拟C盘\光\website"
npm run deploy
```

## 步骤 4：代码仓库长效溯源
将新写的笔记源码推送至 GitHub 进行版本控制：
```bash
cd "d:\虚拟C盘\光\website"
git add .
git commit -m "docs: 新增知识节点 - [文章标题]"
git push origin main
```

完成以上所有步骤后，通知用户网站已经发布成功。
