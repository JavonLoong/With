# 优化 Markdown 文档

对指定的 `.md` 文件执行以下 5 步标准化优化流程。每步完成后直接在文件中修改。

## 使用方法

在对话中说：`/optimize-doc`，然后指定要优化的文件名（或"全部"）。

## 优化步骤

### 第 1 步：信息去重
- 扫描全文，找出**重复表达的同一观点**（可能措辞不同但意思相同）
- 只保留表达最好的一处，删除其余
- 重点检查：跨章节重复（例如中间章节和结尾都在说同一句话）

### 第 2 步：结构精简
- 大段文字能用**表格**更清晰表达的 → 改为表格
- 列表项都很短（<10字）→ 考虑合并为一行
- 删除过渡废话（"接下来我们来看"、"下面说说"等）
- 确保每个 `##` 章节有**明确且不重叠**的主题

### 第 3 步：内容更新
- 检查示例/话术是否与用户**当前阶段**匹配
- 替换过时的场景示例，补充当前适用的场景
- 每个场景保持 **2~3 条**示例，不宜过多

### 第 4 步：格式统一
- 标题层级：`#` 文档标题 → `##` 大章节 → `###` 小节
- emoji/图标风格全文统一（要么 ❌✅，要么 🔴🟢，不混用）
- 引用块 `>` 仅用于话术/示例，解释性文字不用引用块
- 表格对齐、代码块标注语言

### 第 5 步：终审瘦身
- 目标：优化后总行数 ≤ 原文的 **70%**
- 删除"正确的废话"（说了等于没说的句子）
- 确保每一句话都有**信息增量**

## 完成后

- 输出优化前后的行数对比和主要改动说明
- 如果文档与文件夹内其他文件有内容重叠，一并指出
