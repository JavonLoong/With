# 5.6 水波与驻波实验视觉分析系统 (Standing Wave Analysis)

本项目用于自动化分析水波物理实验的高速/常规视频。核心功能包括：绿水边界特征提取（水线提取）、直尺像素标定、频域方向分离分析（Incident/Reflected）、驻波反射率评估，以及波峰间距的高精度测量。

---

## 📁 目录结构 (Directory Structure)

为了保证代码与数据的纯净分离，本项目遵循以下标准结构：

- `src/` : 核心算法模块库。其中 `core_analysis.py` 封装了所有水线提取、驻波诊断的核心算法逻辑。
- `scripts/` : 统一执行入口，目前包含全局运行脚本 `run_all.py`。
- `config.json` : 全局与局部参数配置文件（用于动态调整感兴趣区域 ROI 等）。
- `requirements.txt` : 自动化环境依赖清单。
- `data/` : 数据区（未包含在纯代码压缩包内，需自行建立）
  - `raw/` : 请将待处理的实验视频按组别存放在此处。
  - `processed/` : 未来的报表统一输出存放点。

---

## ⚙️ 环境依赖与安装

推荐使用 **Python 3.9 及以上版本**。
获取本项目代码后，请在项目根目录下打开命令行/终端，执行以下命令安装依赖：

```bash
pip install -r requirements.txt
```

> **依赖说明：**
> * `opencv-python` / `scikit-image` / `imageio`：用于核心视频帧处理与边缘提取。
> * `numpy` / `scipy` / `pandas`：用于大规模矩阵运算、信号平滑、FFT 滤波及寻峰算法。
> * `matplotlib`：用于生成数据可视化与诊断图表。

## 🚀 运行说明与数据准备

### 第一步：准备数据
本程序使用**按文件夹分组**的逻辑读取数据。请你在项目根目录下创建 `data/raw/` 路径，并将实验视频放进去。
**注意：必须为每个工况（如 1.6, 2.6）新建一个子文件夹，视频放在子文件夹内。**

**正确示例：**
```text
data/
└── raw/
    ├── 1.6/
    │   └── 1.6.mp4     <-- 视频文件名随意，程序会自动寻找 mp4
    └── 10.0/
        └── test.mp4
```

### 第二步：一键运行
数据放置完成后，在项目根目录运行以下命令即可全自动跑通所有数据：

```bash
python scripts/run_all.py
```

### 第三步：调整配置 (可选)
如果程序报错“找不到直尺”或“水线提取失误”，**无需修改任何 Python 源码**！
请打开根目录下的 `config.json`，在 `"group_overrides"` 字典中为你报错的工况组别覆写 ROI（感兴趣区域）参数。

```json
"group_overrides": {
    "1.6": {
        "waterline_roi": [0.30, 0.70]  // 表示针对1.6组，将水面搜索范围调至画面高度的 30% ~ 70% 处
    }
}
```

## 📊 结果输出

针对每一个组别（例如 `1.6`），程序运行完毕后，会在其对应的 `data/raw/1.6/` 目录下生成一系列详细报告与结果：

- `standing_wave_summary_cn.csv` : 该工况下所有核心物理量指标的汇总表。
- `standing_wave_cn_report.md` : 自动生成的文字分析报告及研判结果。
- `standing_wave_analysis_data.npz` : 核心算法产生的 `eta(x,t)` 矩阵源数据压缩包。
- `eta_xt_surface_cm.csv` : x-t 维度的水面高度切片表格。
- `*.png` : 大量极具价值的图像诊断文件，包含 **水线提取复核图**、**反射强度时序图**、**入射/反射分离热力图** 等等。
